About

From researching brains to building things with code

  • 2012-2016

    Neuroscience undergrad

    I loved science at school, especially biology and chemistry, so I decided to study neuroscience at Cardiff University. This involved a placement year in my 3rd year at Exeter University working in an electrophysiology lab at a rig like the one in this photo.

  • 2016 - 2019

    First dabble with coding

    I started a job as a research assistant at Cambridge University working on a really exciting project - mapping the brain of a fruit fly! This picture illustrates a section of the fruit fly brain, I would trace neurons through the layered stacks of these images. This is where I got my first taste of what coding was like. Using R and Python I mapped the connections between neurons we traced in the fly brain.

  • 2019 - 2024

    PhD and more coding

    I moved to Edinburgh to do a PhD in Translational Neuroscience funded by the Wellcome Trust. This is where I really fell in love with coding. I worked on various projects which involved cleaning data on thousands of people and running statistical models to look for patterns between mental health outcomes, genetics and blood markers. I also set up a coding club in my department that I still run today.

  • 2023

    App development opportunity

    A colleague approached me to build a tool to help non-technical researchers with conducting complex statistical models. I taught myself R Shiny and delivered a prototype whilst continuing to work on my PhD. This prototype secured us additional funding for me to develop the app further during a 6-month secondment to my PhD.

  • 2024 - Present

    Post-doc & hobby projects

    Since finishing my PhD I took up a post-doctoral position in a different research group in the same department at Edinburgh University. I continue to learn new coding skills in this role as well as explore new technologies in coding side projects.

  • Future Software Developer/Tester

    I'm now looking for software developer and tester opportunites so I can spend more time doing the type of coding that I enjoy the most.

Raspberry Pi Bus Tracker

Skills: Python, Linux

I recently discovered some really cool projects people had made using raspberry pi's and wanted to have a go at this myself. I wanted to create something that would be useful for me and decided on a simple LCD screen that could display the time my next local bus was departing. I also added a message to say when to leave, to walk quickly or to run depending on how many minutes there were until the next bus. If the bus was less than a certain number of minutes away then I would miss it if I left the house at that time, so instead it shows the next available bus I could get.

Data from the Lothian Bus API is fetched, wrangled and output to the LCD screen using a Python script that is scheduled to run every minute in a cron job on the Raspberry Pi.

Cleaning NHS data

Skills: SQL, R

During my PhD I wanted to build up some more of my coding skills and learn SQL. I reached out to one of my colleagues, Dr Katie Marwick, who was planning to work with the SAIL Data Bank (NHS data on the entire Welsh population) on an epidemiological project on post-partum psychosis, to offer help with the code.

This has been a really rewarding experience. I’ve built up my skills in SQL writing complex queries for data with millions of rows, worked inside a Trusted Research Environment (a secure remote desktop where the data is held) and problem solved how to approach and clean many tricky nuances of real world data. Please see the GitHub repo for the SQL code and a schematic diagram illustrating the data cleaning steps.

Coding Club

Set up and lead a coding club - a friendly space to discuss coding problems, create and share resources and learn from each other.

In our department, many individuals encounter similar coding challenges eg. data cleaning/analysis and using high-performance computers. Addressing these challenges together not only makes them less daunting but also enhances efficiency by reducing duplicate efforts through shared code and resources. Additionally, good coding practices are crucial for reproducible science.

The coding club’s primary aim is to foster a friendly, supportive environment for improving coding skills together. We focus on creating and sharing resources that benefit both our research group and the wider research community.

Code Review Workshop

Secured funding from Improving Research Community Builder Award to expand my coding club to Unversity wide audiences

Held a workshop on how to do code review in academia. We started with an engaging introductory presentation that involved audience discussion and participation, covering the fundamentals of code review including what it is, the benefits, how to prepare your code for review and how to review someone else’s code. After a tea/coffee break, attendees put what they had learnt into practice and reviewed R or Python code snippets in pairs. In summary, it was a fantastic first event for this series, with some excellent discussions and participation from our attendees on code review in academia.

git & GitHub webinar

Delivered an online webinar for The Data Lab community

An organiser for the Data Lab events, Marian Dunbar, put a request out on the mailing list for help with organising a webinar on git and GitHub. Myself and Joe Crispell responded to Marian to offer to help. We met online to make a plan, then myself and Joe worked together using git and GitHub to create quarto markdown slides on git and GitHub. We created GitHub issues for the content we wanted to present, used the projects kanban board and did pull requests/code review. The presentation then showed how to use git and GitHub by showing the issues, pull requests etc. that we created.
